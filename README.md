# Qwen Coding Plan for Codex Responses Proxy

> 中文说明在前，English guide follows below.

## 中文说明

这个仓库是一份通用部署方案：在云服务器上部署一个兼容层，让 Codex Desktop / Codex CLI 通过 OpenAI Responses API 调用只提供 OpenAI Chat Completions 兼容接口的阿里百炼 Coding Plan 模型，例如 `qwen3.6-plus`。

核心思路：

```text
Codex Desktop / CLI
  -> go-llm-proxy /v1/responses
  -> 阿里百炼 Coding Plan /v1/chat/completions
  -> qwen3.6-plus
```

Codex 侧可以把模型名写成 `gpt-5.2` 这样的别名，代理服务器再把它转发成真实上游模型 `qwen3.6-plus`。这个别名只是为了让客户端更容易选择和调用，不代表实际请求的是 OpenAI GPT 模型。

## 适用场景

- 你有阿里百炼 Coding Plan 的 API Key。
- 你想让 Codex 使用 `qwen3.6-plus` 这类 Chat Completions 模型。
- 你的 Codex 配置需要 `wire_api = "responses"`。
- 你希望上游 API Key 只保存在服务器上，客户端只拿代理 Key。

## 准备条件

- 一台 Linux 云服务器。
- Docker 和 Docker Compose。
- 一个阿里百炼 Coding Plan API Key。
- 一个你自己生成的代理访问 Key。
- 可选：域名、HTTPS 证书、反向代理。

## 快速部署

1. 克隆仓库：

```bash
git clone https://github.com/YOUR_GITHUB_NAME/qwen-codex-responses-proxy-guide.git
cd qwen-codex-responses-proxy-guide
```

2. 复制配置：

```bash
cp config.example.yaml config.yaml
cp .env.example .env
```

3. 编辑 `config.yaml`，替换这些占位符：

```yaml
YOUR_ALIBABA_CODING_PLAN_API_KEY
YOUR_PROXY_API_KEY
```

推荐自己生成一个强随机代理 Key，例如：

```bash
openssl rand -hex 32
```

4. 启动服务：

```bash
docker compose up -d
```

默认会把服务暴露到服务器的 `3002` 端口，客户端 Base URL 使用：

```text
http://YOUR_SERVER_IP:3002/v1
```

生产环境建议使用 HTTPS，例如：

```text
https://api.example.com/v1
```

## 测试代理

查看模型列表：

```bash
curl http://YOUR_SERVER_IP:3002/v1/models \
```

```bash
curl http://YOUR_SERVER_IP:3002/v1/responses \

  -H "Content-Type: application/json" \
  -d '{
```

## Codex 配置示例

把下面配置放到 Codex 的 `config.toml` 里，按你的环境替换地址和 Key 环境变量名。

```toml
model_provider = "qwen_proxy"
model = "gpt-5.2"
model_reasoning_effort = "high"
web_search = "disabled"

[model_providers.qwen_proxy]
name = "Qwen Proxy"
wire_api = "responses"
base_url = "http://YOUR_SERVER_IP:3002/v1"
env_key = "QWEN_PROXY_API_KEY"
```

启动 Codex 前设置代理 Key：

```bash
export QWEN_PROXY_API_KEY="YOUR_PROXY_API_KEY"
```

Windows PowerShell：

```powershell
$env:QWEN_PROXY_API_KEY="YOUR_PROXY_API_KEY"
```

## 模型别名

示例配置里有两个可用模型名：

- `gpt-5.2`：给 Codex 使用的外部别名。
- `qwen3.6-plus`：保留真实模型名，方便直接测试。

两者都会转发到阿里百炼 Coding Plan 的真实模型 `qwen3.6-plus`。如果你想换别名，只需要修改 `config.yaml` 里的 `models[].name` 和 `keys[].models`。

## 安全建议

- 不要把真实 API Key 写进 Git 仓库。
- 不要把代理端口裸露给全网长期使用，至少要开启代理 Key。
- 生产环境建议加 HTTPS。
- 如果 Key 泄露，立刻在上游平台和代理配置里轮换。
- 这个方案是兼容层，不是 OpenAI 或阿里云官方产品。

## English Guide

This repository provides a generic deployment guide for running a server-side compatibility layer that lets Codex Desktop / Codex CLI call Alibaba Cloud Model Studio Coding Plan models, such as `qwen3.6-plus`, through an OpenAI Responses API compatible endpoint.

Architecture:

```text
Codex Desktop / CLI
  -> go-llm-proxy /v1/responses
  -> Alibaba Coding Plan /v1/chat/completions
  -> qwen3.6-plus
```

Codex can request an alias such as `gpt-5.2`. The proxy rewrites that external model name to the real upstream model `qwen3.6-plus`. The alias is only a client-facing name; it does not mean the upstream model is an OpenAI GPT model.

## Use Cases

- You have an Alibaba Coding Plan API key.
- You want Codex to use a Chat Completions-only model such as `qwen3.6-plus`.
- Your Codex provider must use `wire_api = "responses"`.
- You want the upstream API key to stay on the server, while clients only use a proxy key.

## Requirements

- A Linux server.
- Docker and Docker Compose.
- An Alibaba Coding Plan API key.
- A strong proxy API key generated by you.
- Optional: a domain name, HTTPS certificate, and reverse proxy.

## Quick Start

1. Clone the repository:

```bash
git clone https://github.com/YOUR_GITHUB_NAME/qwen-codex-responses-proxy-guide.git
cd qwen-codex-responses-proxy-guide
```

2. Copy the example files:

```bash
cp config.example.yaml config.yaml
cp .env.example .env
```

3. Edit `config.yaml` and replace:

```yaml
YOUR_ALIBABA_CODING_PLAN_API_KEY
YOUR_PROXY_API_KEY
```

Generate a strong proxy key, for example:

```bash
openssl rand -hex 32
```

4. Start the proxy:

```bash
docker compose up -d
```

By default, the service is exposed on server port `3002`. The client Base URL is:

```text
http://YOUR_SERVER_IP:3002/v1
```

For production, HTTPS is strongly recommended:

```text
https://api.example.com/v1
```

Add a custom provider to your Codex `config.toml` and replace the URL and environment variable as needed.

```toml
model_provider = "qwen_proxy"

[model_providers.qwen_proxy]
name = "Qwen Proxy"
wire_api = "responses"
base_url = "http://YOUR_SERVER_IP:3002/v1"
env_key = "QWEN_PROXY_API_KEY"
```

Set the proxy key before starting Codex:

```bash
export QWEN_PROXY_API_KEY="YOUR_PROXY_API_KEY"
```

Windows PowerShell:

```powershell
$env:QWEN_PROXY_API_KEY="YOUR_PROXY_API_KEY"
```

## Model Aliases

The example config exposes two names:

- `gpt-5.2`: a client-facing alias for Codex.
- `qwen3.6-plus`: the real model name, useful for direct tests.

Both names are forwarded to the upstream Alibaba Coding Plan model `qwen3.6-plus`. To change the alias, edit `models[].name` and `keys[].models` in `config.yaml`.

## Security Notes

- Never commit real API keys to Git.
- Do not expose the proxy publicly without authentication.
- Use HTTPS in production.
- Rotate both upstream and proxy keys immediately if they are leaked.
- This is a compatibility-layer deployment guide, not an official OpenAI or Alibaba Cloud product.
